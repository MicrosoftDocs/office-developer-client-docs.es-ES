# <a name="project-conceptual-and-how-to-articles"></a>Artículos con conceptos e instrucciones de Project

Busque vínculos a información en esta sección sobre las partes principales de la plataforma de Project 2013, para empezar con el desarrollo de CSOM mediante JavaScript y los complementos de panel de tareas .NET Framework en Project Profesional 2013, la interfaz de servidor de Project y el servicio de informes de OData.
  
Project Server 2013 se basa en el marco de servicio web Project Server Interface (PSI) introducido en Office Project Server 2007. La Project Server 2013 se ha creado con el .NET Framework 4. Project Web App requiere SharePoint Server 2013 como base para la funcionalidad de nuevas características, como el modelo de objetos del lado cliente (CSOM) para obtener acceso a Project Server en la nube, la versión 4 de Windows Workflow Foundation (WF4) y el servicio OData para las tablas de base de datos de informes. Project Profesional 2013 puede usar el nuevo panel de tareas Office complementos.

- Las actualizaciones para desarrolladores de [Project 2013](updates-for-developers-in-project-2013.md) describen las características de extensibilidad para el desarrollo Project y enumera las características en desuso que no deben usarse para el desarrollo nuevo. 
  
- Project arquitectura y programación de [server 2013](project-server-2013-architecture-and-programmability.md) describe las partes principales de la plataforma de Project 2013, incluidos los clientes Project y Project Server. 
  
- El modelo de objetos del lado cliente [(CSOM) para Project 2013](client-side-object-model-csom-for-project-2013.md) incluye temas para empezar con el desarrollo de CSOM mediante JavaScript y el .NET Framework. 
  
- [Project programación de cliente incluye](project-client-programming.md) una introducción a los complementos de panel de tareas en Project Profesional 2013. Un SDK independiente incluye artículos y referencias para complementos en Project y los demás clientes Office 2013. Para obtener más información, [consulte la Office para desarrolladores de complementos.](https://docs.microsoft.com/office/dev/add-ins/overview/office-add-ins) 
  
- [Project](project-programming-tasks.md) tareas de programación muestra cómo desarrollar complementos para Project 2013, crear flujos de trabajo para la administración de la demanda, personalizar la cinta de opciones de Project Web App, crear elementos web y mucho más. 
  
- [Project de programación de 2013](project-2013-programming-references.md) incluye información sobre el desarrollo con la interfaz de servidor (PSI) de Project, los códigos de error y el servicio ProjectData OData.
