# <a name="project-conceptual-and-how-to-articles"></a>Artículos con conceptos e instrucciones de Project

Busque vínculos a información en esta sección sobre las partes principales de la plataforma De Project 2013, introducción al desarrollo de CSOM mediante JavaScript y .NET Framework, complementos de panel de tareas en Project Professional 2013, Project Server Interface y el servicio de informes de OData.
  
Project Server 2013 se basa en el marco de servicio web de Project Server Interface (PSI) introducido en Office Project Server 2007. La plataforma de Project Server 2013 se ha creado con .NET Framework 4. Project Web App requiere SharePoint Server 2013 como base para la funcionalidad de nuevas características, como el modelo de objetos del lado cliente (CSOM) para obtener acceso a Project Server en la nube, la versión 4 de Windows Workflow Foundation (WF4) y el servicio OData para las tablas de la base de datos de informes. Project Professional 2013 puede usar el nuevo panel de tareas Complementos de Office.

- [Las actualizaciones para desarrolladores de Project 2013](updates-for-developers-in-project-2013.md) describen las características de extensibilidad para el desarrollo de Project y enumeran las características en desuso que no deben usarse para el desarrollo nuevo. 
  
- La arquitectura y programación de [Project Server 2013](project-server-2013-architecture-and-programmability.md) describe las partes principales de la plataforma de Project 2013, incluidos los clientes de Project y Project Server. 
  
- El modelo de objetos de [cliente (CSOM) para Project 2013](client-side-object-model-csom-for-project-2013.md) incluye temas para empezar a desarrollar CSOM mediante JavaScript y .NET Framework. 
  
- [La programación del](project-client-programming.md) cliente de Project incluye una introducción a los complementos de panel de tareas en Project Professional 2013. Un SDK independiente incluye artículos y referencias para complementos en Project y los demás clientes de Office 2013. Para obtener más información, consulte la [documentación para desarrolladores de complementos](https://docs.microsoft.com/office/dev/add-ins/overview/office-add-ins) de Office. 
  
- Las tareas de programación de [Project](project-programming-tasks.md) muestran cómo desarrollar complementos para Project 2013, crear flujos de trabajo para la administración de demandas, personalizar la cinta de Project Web App, crear elementos web y mucho más. 
  
- Las referencias de programación de [Project 2013](project-2013-programming-references.md) incluyen información sobre el desarrollo con Project Server Interface (PSI), códigos de error y el servicio ProjectData OData.
