# <a name="project-conceptual-and-how-to-articles"></a>Artículos con conceptos e instrucciones de Project

Encuentre vínculos a la información de esta sección sobre las partes principales de la plataforma Project 2013, introducción al desarrollo de CSOM con JavaScript y .NET Framework, Complementos de panel de tareas en Project Professional 2013, la interfaz de Project Server y el Servicio de informes de OData.
  
Project Server 2013 se basa en el marco del servicio Web de Project Server Interface (PSI) que se ha incluido en Office Project Server 2007. La plataforma de Project Server 2013 se basa en .NET Framework 4. Project Web App requiere SharePoint Server 2013 como base para la funcionalidad de las nuevas características, como el modelo de objetos del lado cliente (CSOM) para el acceso a Project Server en la nube, la versión 4 de Windows Workflow Foundation (WF4) y el servicio OData para el tablas de la base de datos de informes. Project Professional 2013 puede usar los nuevos complementos de Office de panel de tareas.

- [Updates for Developers in project 2013](updates-for-developers-in-project-2013.md) describe las características de extensibilidad para el desarrollo de proyectos y enumera las características en desuso que no se deben usar para el desarrollo nuevo. 
  
- La [arquitectura y programación de Project Server 2013](project-server-2013-architecture-and-programmability.md) describe las partes principales de la plataforma de Project 2013, incluidos los clientes de Project y Project Server. 
  
- El [modelo de objetos de cliente (CSOM) para el proyecto 2013](client-side-object-model-csom-for-project-2013.md) incluye temas para empezar a trabajar con el desarrollo de CSOM mediante JavaScript y .NET Framework. 
  
- La [programación del cliente de Project](project-client-programming.md) incluye una introducción a los complementos de panel de tareas en Project Professional 2013. Un SDK independiente incluye artículos y referencias para complementos en Project y los otros clientes de Office 2013. Para obtener más información, vea la documentación para desarrolladores de complementos de [Office](https://docs.microsoft.com/office/dev/add-ins/overview/office-add-ins) . 
  
- [Project Programming Tasks](project-programming-tasks.md) muestra cómo desarrollar complementos para Project 2013, crear flujos de trabajo para la administración de propuestas, personalizar la cinta de Project Web App, crear elementos Web y mucho más.
