# <a name="pidtagsentrepresentingsmtpaddress-canonical-property"></a>Propiedad canónica PidTagSentRepresentingSmtpAddress

  
  
**Se aplica a**: Outlook 2013 | Outlook 2016 
  
Contiene la dirección de correo electrónico del Protocolo simple de transporte de correo (SMTP) para el usuario de mensajería representado por el remitente.
  
|||
|:-----|:-----|
|Propiedades asociadas:  <br/> |PR_SENT_REPRESENTING_SMTP_ADDRESS, PR_SENT_REPRESENTING_SMTP_ADDRESS_A, PR_SENT_REPRESENTING_SMTP_ADDRESS_W  <br/> |
|Identificador:  <br/> |0x5D02  <br/> |
|Tipo de datos:  <br/> |PT_UNICODE, PT_STRING8  <br/> |
|Área:  <br/> |Address  <br/> |
   
## <a name="remarks"></a>Comentarios

Esta propiedad es un ejemplo de las propiedades de dirección del usuario de mensajería que representa el remitente. Cuando una aplicación cliente envía un mensaje en nombre de otro cliente, debe establecer todas las propiedades de remitente representadas en los valores de ese cliente. Normalmente, un usuario de mensajería que envía en su propio nombre deja sin conjunto las propiedades del remitente representado.
  
El proveedor de transporte saliente siempre debe dejar esta propiedad sin cambios si la ha establecido el cliente de envío. Si no se establece, el proveedor de transporte debe establecerla en la propiedad **PR_SENDER_SMTP_ADDRESS** ([PidTagSenderSmtpAddress](pidtagsendersmtpaddress-canonical-property.md)) en la copia saliente del mensaje y dejarla sin establecer en la copia local.
