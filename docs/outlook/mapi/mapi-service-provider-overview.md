# <a name="mapi-service-provider-overview"></a><span data-ttu-id="d36fc-103">Introducción al proveedor de servicios MAPI</span><span class="sxs-lookup"><span data-stu-id="d36fc-103">MAPI Service Provider Overview</span></span>

  
  
<span data-ttu-id="d36fc-104">**Se aplica a**: Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="d36fc-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="d36fc-105">Entre el subsistema MAPI y los sistemas de mensajería se encuentran los distintos proveedores de servicios.</span><span class="sxs-lookup"><span data-stu-id="d36fc-105">Between the MAPI subsystem and the messaging systems are the various service providers.</span></span> <span data-ttu-id="d36fc-106">Los proveedores de servicios son como controladores que conectan aplicaciones cliente MAPI a un sistema de mensajería subyacente.</span><span class="sxs-lookup"><span data-stu-id="d36fc-106">Service providers are like drivers that connect MAPI client applications to an underlying messaging system.</span></span> <span data-ttu-id="d36fc-107">Hay tres tipos de proveedores: proveedores de almacén de mensajes, libreta de direcciones o proveedores de directorios y proveedores de transporte de mensajes.</span><span class="sxs-lookup"><span data-stu-id="d36fc-107">There are three types of providers: message store providers, address book or directory providers, and message transport providers.</span></span> <span data-ttu-id="d36fc-108">MAPI admite cada tipo de servicio de forma independiente, lo que permite a un proveedor ofrecer uno o más proveedores de servicios personalizados.</span><span class="sxs-lookup"><span data-stu-id="d36fc-108">MAPI supports each type of service independently, enabling a vendor to offer one or more custom service providers.</span></span> <span data-ttu-id="d36fc-109">Por ejemplo, es posible que un proveedor desee crear un proveedor de libreta de direcciones que use un directorio de libreta de teléfonos corporativo de empleados o crear un proveedor de almacén de mensajes que use una base de datos existente.</span><span class="sxs-lookup"><span data-stu-id="d36fc-109">For example, a vendor might want to create an address book provider that uses a corporate telephone book directory of employees or to create a message store provider that uses an existing database.</span></span>
  
<span data-ttu-id="d36fc-110">Los proveedores de servicios suelen estar escritos para sistemas de mensajería específicos por desarrolladores de software que tienen conocimientos especializados o experiencia con un sistema en particular.</span><span class="sxs-lookup"><span data-stu-id="d36fc-110">Service providers are typically written for specific messaging systems by software developers who have specialized knowledge or experience with a particular system.</span></span> <span data-ttu-id="d36fc-111">Por ejemplo, Microsoft Outlook 2013 y Microsoft Outlook 2010 Mobile Services un proveedor de libreta de direcciones para exponer una libreta de direcciones móvil en Outlook.</span><span class="sxs-lookup"><span data-stu-id="d36fc-111">For example, the Microsoft Outlook 2013 and Microsoft Outlook 2010 Mobile Services use an address book provider to expose a mobile address book in Outlook.</span></span> 
  
<span data-ttu-id="d36fc-112">MAPI presenta aplicaciones cliente con una vista unificada de la información del proveedor de transporte y libreta de direcciones.</span><span class="sxs-lookup"><span data-stu-id="d36fc-112">MAPI presents client applications with a unified view of address book and transport provider information.</span></span> <span data-ttu-id="d36fc-113">Este enfoque integrado impide que la aplicación cliente tenga que asignar datos al proveedor adecuado.</span><span class="sxs-lookup"><span data-stu-id="d36fc-113">This integrated approach prevents the client application from having to map data to the appropriate provider.</span></span> <span data-ttu-id="d36fc-114">También evita que el usuario tenga que negociar entre varios esquemas de direccionamiento de libreta de direcciones y proveedor de transporte.</span><span class="sxs-lookup"><span data-stu-id="d36fc-114">It also prevents the user from having to negotiate among multiple address book and transport provider addressing schemes.</span></span> <span data-ttu-id="d36fc-115">Sin embargo, la información del proveedor del almacén de mensajes no está unificada y los clientes que usan varios proveedores de almacén de mensajes son responsables de controlarlos individualmente.</span><span class="sxs-lookup"><span data-stu-id="d36fc-115">Message store provider information, however, is not unified, and clients that use multiple message store providers are responsible for handling them individually.</span></span>
  
<span data-ttu-id="d36fc-116">Los proveedores de servicios trabajan con MAPI para crear y enviar mensajes de la siguiente manera: los mensajes se crean mediante un formulario adecuado para el tipo o clase específicos de mensaje.</span><span class="sxs-lookup"><span data-stu-id="d36fc-116">The service providers work with MAPI to create and send messages in the following way: messages are created by using a form that is appropriate for the specific type, or class, of message.</span></span> <span data-ttu-id="d36fc-117">Muchos mensajes se realizan con el formulario de nota estándar que viene con el subsistema MAPI, ya sea por el usuario de una aplicación cliente o mediante programación sin interacción del usuario.</span><span class="sxs-lookup"><span data-stu-id="d36fc-117">Many messages are made with the standard note form that comes with the MAPI subsystem, either by the user of a client application or programmatically without user interaction.</span></span> <span data-ttu-id="d36fc-118">El mensaje completado se dirige a uno o varios destinatarios: un usuario o grupo de usuarios designados para recibir el mensaje.</span><span class="sxs-lookup"><span data-stu-id="d36fc-118">The completed message is addressed to one or more recipients — a user or group of users designated to receive the message.</span></span> <span data-ttu-id="d36fc-119">Un destinatario puede o no tener una entrada en un directorio que uno de los proveedores de libreta de direcciones instalados posee.</span><span class="sxs-lookup"><span data-stu-id="d36fc-119">A recipient might or might not have an entry in a directory that one of the installed address book providers owns.</span></span> <span data-ttu-id="d36fc-120">Los destinatarios que no están asociados con un proveedor de libreta de direcciones instalado se denominan destinatarios personalizados o direcciones únicas.</span><span class="sxs-lookup"><span data-stu-id="d36fc-120">Recipients that are not associated with an installed address book provider are called custom recipients or one-off addresses.</span></span> <span data-ttu-id="d36fc-121">Una dirección de uso único puede ser temporal y solo dura hasta que se envía el mensaje.</span><span class="sxs-lookup"><span data-stu-id="d36fc-121">A one-off address can be temporary, lasting only until the message is submitted.</span></span> 
  
<span data-ttu-id="d36fc-122">Cuando la aplicación cliente envía el mensaje, el proveedor del almacén de mensajes comprueba que cada destinatario tiene una dirección única y válida y que el mensaje tiene toda la información necesaria para la transmisión.</span><span class="sxs-lookup"><span data-stu-id="d36fc-122">When the client application sends the message, the message store provider checks that each recipient has a unique and valid address and that the message has all of the information necessary for transmission.</span></span> <span data-ttu-id="d36fc-123">Si hay una pregunta sobre un destinatario (por ejemplo, cuando hay varios destinatarios con el mismo nombre), un proveedor de libreta de direcciones se encarga de resolver la ambigüedad.</span><span class="sxs-lookup"><span data-stu-id="d36fc-123">If there is a question about a recipient (for example, when there are multiple recipients with the same name), an address book provider takes care of resolving the ambiguity.</span></span> <span data-ttu-id="d36fc-124">A continuación, el mensaje se coloca en la cola de salida.</span><span class="sxs-lookup"><span data-stu-id="d36fc-124">The message is then placed in the outbound queue.</span></span>
