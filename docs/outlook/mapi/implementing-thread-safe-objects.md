<span data-ttu-id="264f1-107">Un cliente puede implementar una devolución de llamada de notificación segura para subprocesos llamando a la utilidad MAPI [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span><span class="sxs-lookup"><span data-stu-id="264f1-107">A client can implement a thread-safe notification callback by calling the MAPI utility [HrThisThreadAdviseSink](hrthisthreadadvisesink.md).</span></span> <span data-ttu-id="264f1-108">**HrThisThreadAdviseSink** transforma un receptor de aviso no seguro para subprocesos en uno seguro para subprocesos.</span><span class="sxs-lookup"><span data-stu-id="264f1-108">**HrThisThreadAdviseSink** transforms a non-thread-safe advise sink into a thread-safe one.</span></span> <span data-ttu-id="264f1-109">Para las devoluciones de llamada de progreso, no existe dicha utilidad.</span><span class="sxs-lookup"><span data-stu-id="264f1-109">For progress callbacks, there is no such utility.</span></span> <span data-ttu-id="264f1-110">Un cliente puede elegir usar el objeto de progreso seguro para subprocesos MAPI o crear uno manualmente.</span><span class="sxs-lookup"><span data-stu-id="264f1-110">A client can choose to use the MAPI thread-safe progress object or create one manually.</span></span> 
  
<span data-ttu-id="264f1-111">Un objeto seguro para subprocesos también puede o no ser consciente de subprocesos.</span><span class="sxs-lookup"><span data-stu-id="264f1-111">A thread-safe object might or might not also be thread-aware.</span></span> <span data-ttu-id="264f1-112">Un objeto para subprocesos mantiene un contexto independiente para cada subproceso que lo está usando.</span><span class="sxs-lookup"><span data-stu-id="264f1-112">A thread-aware object maintains a separate context for every thread that is using it.</span></span> <span data-ttu-id="264f1-113">Los proveedores de servicios no son necesarios para admitir el reconocimiento de subprocesos en sus objetos seguros para subprocesos, aunque la compatibilidad con el reconocimiento de subprocesos puede ser útil en algunas situaciones.</span><span class="sxs-lookup"><span data-stu-id="264f1-113">Service providers are not required to support thread-awareness in their thread-safe objects, although supporting thread-awareness can be useful in some situations.</span></span> <span data-ttu-id="264f1-114">Dos tablas MAPI siempre proporcionan su propio contexto por definición.</span><span class="sxs-lookup"><span data-stu-id="264f1-114">Two MAPI tables always provide their own context by definition.</span></span> <span data-ttu-id="264f1-115">Una tabla usada en subprocesos diferentes no proporciona ni debe proporcionar un contexto único.</span><span class="sxs-lookup"><span data-stu-id="264f1-115">One table used on different threads does not and should not provide unique context.</span></span>
  
<span data-ttu-id="264f1-116">Un cliente puede elegir entre recibir notificaciones en el mismo subproceso que se usó para la llamada **MAPIInitialize,** en el mismo subproceso que se usó para la llamada **Advise** o en un subproceso independiente propiedad de MAPI.</span><span class="sxs-lookup"><span data-stu-id="264f1-116">A client can choose between receiving notifications on the same thread that was used for the **MAPIInitialize** call, on the same thread that was used for the **Advise** call, or on a separate thread owned by MAPI.</span></span> <span data-ttu-id="264f1-117">Para asegurarse de que las notificaciones lleguen al mismo subproceso que se usó para llamar a **MAPIInitialize,** un cliente llama a [MAPIInitialize](mapiinitialize.md) y pasa cero en el miembro **ulFlags** de la [estructura MAPIINIT_0](mapiinit_0.md) web.</span><span class="sxs-lookup"><span data-stu-id="264f1-117">To ensure that notifications arrive on the same thread that was used to call **MAPIInitialize**, a client calls [MAPIInitialize](mapiinitialize.md) and passes zero in the **ulFlags** member of the [MAPIINIT_0](mapiinit_0.md) structure.</span></span> <span data-ttu-id="264f1-118">A continuación, las notificaciones se entregan durante el bucle de mensajes principal.</span><span class="sxs-lookup"><span data-stu-id="264f1-118">Notifications are then delivered during the main message loop.</span></span> 
  
<span data-ttu-id="264f1-119">Para recibir notificaciones en el subproceso propiedad de MAPI, un cliente llama a **MAPIInitialize** con el **miembro ulFlags** de la estructura **MAPIINIT_0** establecida en MAPI_MULTITHREAD_NOTIFICATIONS.</span><span class="sxs-lookup"><span data-stu-id="264f1-119">To receive notifications on the MAPI-owned thread, a client calls **MAPIInitialize** with the **ulFlags** member of the **MAPIINIT_0** structure set to MAPI_MULTITHREAD_NOTIFICATIONS.</span></span> <span data-ttu-id="264f1-120">La **llamada advise** se realiza con el objeto receptor de aviso del cliente en lugar de una versión ajustada.</span><span class="sxs-lookup"><span data-stu-id="264f1-120">The **Advise** call is made with the client's advise sink object rather than a wrapped version.</span></span> 
  
<span data-ttu-id="264f1-121">Para asegurarse de que las notificaciones lleguen al mismo subproceso que se usó para llamar a **Advise**, un cliente llama a [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) y pasa el receptor de avisos ajustado recién creado a **Advise** en lugar del receptor de avisos original.</span><span class="sxs-lookup"><span data-stu-id="264f1-121">To ensure that notifications arrive on the same thread that was used to call **Advise**, a client calls [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) and passes the newly created wrapped advise sink to **Advise** rather than the original advise sink.</span></span> <span data-ttu-id="264f1-122">**Se puede llamar a MAPIInitialize** con cualquier valor de marca.</span><span class="sxs-lookup"><span data-stu-id="264f1-122">**MAPIInitialize** can be called with either flag value.</span></span>
