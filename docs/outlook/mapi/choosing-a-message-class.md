- <span data-ttu-id="90453-108">La cadena debe cumplir todas las convenciones descritas en la documentación para **la propiedad PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="90453-108">The string should satisfy all the conventions described in the documentation for the **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)) property.</span></span> <span data-ttu-id="90453-109">Es importante destacar que la cadena debe estar compuesta por completo por caracteres ANSI y tener menos de 256 caracteres.</span><span class="sxs-lookup"><span data-stu-id="90453-109">Importantly, the string must be composed entirely of ANSI characters and be less than 256 characters long.</span></span>
    
- <span data-ttu-id="90453-110">Si el servidor de formulario se deriva de un servidor de formulario existente o es una extensión de un servidor de formulario existente, la cadena de clase de mensaje debe formarse agregando un punto y otra palabra a la cadena de clase de mensaje del servidor de formulario en el que se basa el formulario.</span><span class="sxs-lookup"><span data-stu-id="90453-110">If your form server is derived from an existing form server or is an extension of an existing form server, your message class string should be formed by adding a period and another word to the message class string of the form server that your form is based on.</span></span> <span data-ttu-id="90453-111">Por ejemplo, puede que desee implementar un formulario para reprogramar una reunión y el formulario se basa en un formulario existente para programar reuniones.</span><span class="sxs-lookup"><span data-stu-id="90453-111">For example, you might want to implement a form to reschedule a meeting, and your form is based on an existing form for scheduling meetings.</span></span> <span data-ttu-id="90453-112">Si la cadena de clase de mensaje del formulario de programación de reuniones es "IPM. - <span data-ttu-id="90453-113">Si el formulario no se basa en ningún formulario existente, la cadena de clase de mensaje debe comenzar con el "IPM".</span><span class="sxs-lookup"><span data-stu-id="90453-113">If your form is not based on any existing form, your message class string should still begin with either the "IPM."</span></span> <span data-ttu-id="90453-114">o "IPC".</span><span class="sxs-lookup"><span data-stu-id="90453-114">or "IPC."</span></span> <span data-ttu-id="90453-115">en función de si el formulario está pensado para ser recibido por una persona o por otra aplicación.</span><span class="sxs-lookup"><span data-stu-id="90453-115">prefix, depending on whether the form is intended to be received by a person or by another application.</span></span> <span data-ttu-id="90453-116">"IPM".</span><span class="sxs-lookup"><span data-stu-id="90453-116">"IPM."</span></span> <span data-ttu-id="90453-117">designa un mensaje interpersonal que normalmente termina en la Bandeja de entrada de un usuario y "IPC".</span><span class="sxs-lookup"><span data-stu-id="90453-117">designates an interpersonal message that usually ends up in a user's Inbox, and "IPC."</span></span> <span data-ttu-id="90453-118">designa un mensaje de comunicación entre procesos que normalmente no se entrega a la Bandeja de entrada de un usuario.</span><span class="sxs-lookup"><span data-stu-id="90453-118">designates an interprocess communication message that is not typically delivered to a user's Inbox.</span></span>
    
- <span data-ttu-id="90453-119">Si la clase de mensaje está pensada para ser legible para el usuario, la cadena de clase de mensaje debe comenzar con "IPM".</span><span class="sxs-lookup"><span data-stu-id="90453-119">If your message class is intended to be human-readable, the message class string should start with "IPM."</span></span> <span data-ttu-id="90453-120">Por lo general, una clase de mensaje se considera legible para las personas si usa propiedades que contienen datos de texto sin formato, HTML o formato de texto enriquecido (RTF).</span><span class="sxs-lookup"><span data-stu-id="90453-120">A message class is generally considered human-readable if it uses any properties that contain plain text, HTML, or Rich Text Format (RTF) data.</span></span> <span data-ttu-id="90453-121">Si el formulario usa la **PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)), casi con toda seguridad debe usar un "IPM".</span><span class="sxs-lookup"><span data-stu-id="90453-121">If your form uses the **PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)) property, it should almost certainly use an "IPM."</span></span> <span data-ttu-id="90453-122">cadena de clase de mensaje.</span><span class="sxs-lookup"><span data-stu-id="90453-122">message class string.</span></span> <span data-ttu-id="90453-123">Por ejemplo, si va a implementar un formulario para los pedidos de compra y su organización requiere que un administrador apruebe los pedidos de compra, la cadena de clase de mensaje podría ser "IPM. Purchase_Order".</span><span class="sxs-lookup"><span data-stu-id="90453-123">For example, if you are implementing a form for purchase orders, and your organization requires that purchase orders be approved by a manager, your message class string could be "IPM.Purchase_Order".</span></span> <span data-ttu-id="90453-124">Los formularios diseñados para su uso con carpetas públicas o aplicaciones de carpetas públicas suelen considerarse interpersonales, ya que los leen las personas aunque no se dirigen realmente a la dirección de correo electrónico de ninguna persona.</span><span class="sxs-lookup"><span data-stu-id="90453-124">Forms that are designed for use with public folders or public folder applications are typically considered to be interpersonal because they are read by people even though they are not actually addressed to any person's email address.</span></span> <span data-ttu-id="90453-125">El prefijo típico para las clases de mensajes de carpetas públicas es "IPM. - <span data-ttu-id="90453-126">Si la clase de mensaje está pensada para ser recibida por otra aplicación en lugar de por una persona, la cadena de clase de mensaje debe comenzar con "IPC".</span><span class="sxs-lookup"><span data-stu-id="90453-126">If your message class is intended to be received by another application instead of by a person, the message class string should start with "IPC."</span></span> <span data-ttu-id="90453-127">Por ejemplo, si está implementando un formulario que permite a los usuarios suscribirse automáticamente a listas de correo, la cadena de clase de mensaje podría ser "IPC. <span data-ttu-id="90453-129">La cadena de clase de mensaje debe colocarse en la sección **[Descripción]** del archivo de configuración del formulario, en la **entrada MessageClass,** similar a la siguiente:</span><span class="sxs-lookup"><span data-stu-id="90453-129">The message class string should be put in the **[Description]** section of the form configuration file, in the **MessageClass** entry, similar to the following:</span></span>
