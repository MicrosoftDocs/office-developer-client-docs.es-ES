<span data-ttu-id="5af2e-105">Como se describe en Clases de mensajes [MAPI,](mapi-message-classes.md)las clases de mensajes son importantes para establecer la relación entre los tipos de mensajes personalizados y, por extensión, entre los propios servidores de formulario.</span><span class="sxs-lookup"><span data-stu-id="5af2e-105">As described in [MAPI Message Classes](mapi-message-classes.md), message classes are important for establishing the relationship between types of custom messages and, by extension, between form servers themselves.</span></span> <span data-ttu-id="5af2e-106">Afortunadamente, elegir una cadena de clase de mensaje es bastante sencillo.</span><span class="sxs-lookup"><span data-stu-id="5af2e-106">Fortunately, choosing a message class string is fairly simple.</span></span> <span data-ttu-id="5af2e-107">La cadena de clase de mensaje de una clase de mensaje es una cadena arbitraria, pero debe usar las convenciones siguientes:</span><span class="sxs-lookup"><span data-stu-id="5af2e-107">The message class string of a message class is an arbitrary string, but it should use the following conventions:</span></span>
  
- <span data-ttu-id="5af2e-108">La cadena debe satisfacer todas las convenciones descritas en la documentación de la **propiedad PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)).</span><span class="sxs-lookup"><span data-stu-id="5af2e-108">The string should satisfy all the conventions described in the documentation for the **PR_MESSAGE_CLASS** ([PidTagMessageClass](pidtagmessageclass-canonical-property.md)) property.</span></span> <span data-ttu-id="5af2e-109">Es importante destacar que la cadena debe estar compuesta por completo de caracteres ANSI y tener menos de 256 caracteres de longitud.</span><span class="sxs-lookup"><span data-stu-id="5af2e-109">Importantly, the string must be composed entirely of ANSI characters and be less than 256 characters long.</span></span>
    
- <span data-ttu-id="5af2e-110">Si el servidor de formularios deriva de un servidor de formulario existente o es una extensión de un servidor de formulario existente, la cadena de clase de mensaje debe formarse agregando un punto y otra palabra a la cadena de clase de mensaje del servidor de formulario en el que se basa el formulario.</span><span class="sxs-lookup"><span data-stu-id="5af2e-110">If your form server is derived from an existing form server or is an extension of an existing form server, your message class string should be formed by adding a period and another word to the message class string of the form server that your form is based on.</span></span> <span data-ttu-id="5af2e-111">Por ejemplo, es posible que desee implementar un formulario para reprogramar una reunión y el formulario se basa en un formulario existente para programar reuniones.</span><span class="sxs-lookup"><span data-stu-id="5af2e-111">For example, you might want to implement a form to reschedule a meeting, and your form is based on an existing form for scheduling meetings.</span></span> <span data-ttu-id="5af2e-112">Si la cadena de clase de mensaje del formulario de programación de reuniones es "IPM. Meeting.Reschedule".</span><span class="sxs-lookup"><span data-stu-id="5af2e-112">If the meeting scheduling form's message class string is "IPM.Meeting", your message class string could be "IPM.Meeting.Reschedule".</span></span>
    
- <span data-ttu-id="5af2e-113">Si el formulario no se basa en ningún formulario existente, la cadena de clase de mensaje debe comenzar por "IPM".</span><span class="sxs-lookup"><span data-stu-id="5af2e-113">If your form is not based on any existing form, your message class string should still begin with either the "IPM."</span></span> <span data-ttu-id="5af2e-114">o "IPC".</span><span class="sxs-lookup"><span data-stu-id="5af2e-114">or "IPC."</span></span> <span data-ttu-id="5af2e-115">prefijo, en función de si el formulario está pensado para ser recibido por una persona o por otra aplicación.</span><span class="sxs-lookup"><span data-stu-id="5af2e-115">prefix, depending on whether the form is intended to be received by a person or by another application.</span></span> <span data-ttu-id="5af2e-116">"IPM".</span><span class="sxs-lookup"><span data-stu-id="5af2e-116">"IPM."</span></span> <span data-ttu-id="5af2e-117">designa un mensaje interpersonal que suele terminar en la Bandeja de entrada de un usuario y "IPC".</span><span class="sxs-lookup"><span data-stu-id="5af2e-117">designates an interpersonal message that usually ends up in a user's Inbox, and "IPC."</span></span> <span data-ttu-id="5af2e-118">designa un mensaje de comunicación entre procesos que normalmente no se entrega a la Bandeja de entrada de un usuario.</span><span class="sxs-lookup"><span data-stu-id="5af2e-118">designates an interprocess communication message that is not typically delivered to a user's Inbox.</span></span>
    
- <span data-ttu-id="5af2e-119">Si la clase de mensaje está diseñada para ser legible, la cadena de clase de mensaje debe comenzar por "IPM".</span><span class="sxs-lookup"><span data-stu-id="5af2e-119">If your message class is intended to be human-readable, the message class string should start with "IPM."</span></span> <span data-ttu-id="5af2e-120">Por lo general, una clase de mensaje se considera legible para personas si usa propiedades que contienen datos de texto sin formato, HTML o formato de texto enriquecido (RTF).</span><span class="sxs-lookup"><span data-stu-id="5af2e-120">A message class is generally considered human-readable if it uses any properties that contain plain text, HTML, or Rich Text Format (RTF) data.</span></span> <span data-ttu-id="5af2e-121">Si el formulario usa la **propiedad PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)), casi seguro que debe usar un "IPM".</span><span class="sxs-lookup"><span data-stu-id="5af2e-121">If your form uses the **PR_BODY** ([PidTagBody](pidtagbody-canonical-property.md)) property, it should almost certainly use an "IPM."</span></span> <span data-ttu-id="5af2e-122">cadena de clase message.</span><span class="sxs-lookup"><span data-stu-id="5af2e-122">message class string.</span></span> <span data-ttu-id="5af2e-123">Por ejemplo, si va a implementar un formulario para pedidos de compra y su organización requiere que un administrador apruebe los pedidos de compra, la cadena de clase de mensaje podría ser "IPM. Purchase_Order".</span><span class="sxs-lookup"><span data-stu-id="5af2e-123">For example, if you are implementing a form for purchase orders, and your organization requires that purchase orders be approved by a manager, your message class string could be "IPM.Purchase_Order".</span></span> <span data-ttu-id="5af2e-124">Los formularios diseñados para su uso con carpetas públicas o aplicaciones de carpetas públicas suelen considerarse interpersonales porque son leídos por las personas, aunque en realidad no están dirigidos a la dirección de correo electrónico de ninguna persona.</span><span class="sxs-lookup"><span data-stu-id="5af2e-124">Forms that are designed for use with public folders or public folder applications are typically considered to be interpersonal because they are read by people even though they are not actually addressed to any person's email address.</span></span> <span data-ttu-id="5af2e-125">El prefijo típico de las clases de mensajes de carpetas públicas es "IPM. - <span data-ttu-id="5af2e-126">Si la clase de mensaje está diseñada para ser recibida por otra aplicación en lugar de por una persona, la cadena de clase de mensaje debe comenzar por "IPC".</span><span class="sxs-lookup"><span data-stu-id="5af2e-126">If your message class is intended to be received by another application instead of by a person, the message class string should start with "IPC."</span></span> <span data-ttu-id="5af2e-127">Por ejemplo, si va a implementar un formulario que permite a las personas suscribirse automáticamente a listas de correo, la cadena de clase de mensaje podría ser "IPC. <span data-ttu-id="5af2e-129">La cadena de clase de mensaje debe colocarse en la sección **[Descripción]** del archivo de configuración del formulario, en la **entrada MessageClass,** de forma similar a la siguiente:</span><span class="sxs-lookup"><span data-stu-id="5af2e-129">The message class string should be put in the **[Description]** section of the form configuration file, in the **MessageClass** entry, similar to the following:</span></span>

<span data-ttu-id="5af2e-133">Por supuesto, el identificador de clase será casi distinto.</span><span class="sxs-lookup"><span data-stu-id="5af2e-133">Your class identifier will almost certainly be different, of course.</span></span> <span data-ttu-id="5af2e-134">Para obtener más información, vea [Creating a Form Configuration File](creating-a-form-configuration-file.md).</span><span class="sxs-lookup"><span data-stu-id="5af2e-134">For more information, see [Creating a Form Configuration File](creating-a-form-configuration-file.md).</span></span>
