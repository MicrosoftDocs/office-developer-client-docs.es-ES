# <a name="guidelines-for-properly-displaying-activities"></a><span data-ttu-id="aa09a-103">Directrices para mostrar las actividades correctamente</span><span class="sxs-lookup"><span data-stu-id="aa09a-103">Guidelines for properly displaying activities</span></span>

<span data-ttu-id="aa09a-104">Proveedores de Outlook Social Connector (OSC) pueden establecer la **getActivities** y elementos de **dynamicActivitiesLookupEx** tener el OSC almacenan elementos de actividad en la memoria.</span><span class="sxs-lookup"><span data-stu-id="aa09a-104">Outlook Social Connector (OSC) providers can set the **getActivities** and **dynamicActivitiesLookupEx** elements to have the OSC store activity items in memory.</span></span> <span data-ttu-id="aa09a-105">En este tema se describe la extensibilidad del proveedor OSC fuentes de las API que llama el OSC para obtener o actualizar las actividades y detalles de propietario de la actividad, si el proveedor de OSC admite la sincronización de actividad de la red social.</span><span class="sxs-lookup"><span data-stu-id="aa09a-105">This topic describes the OSC provider extensibility APIs that the OSC calls to get or refresh activities and activity owner details, if the OSC provider supports synchronizing activity feeds from the social network.</span></span> <span data-ttu-id="aa09a-106">El tema también enumera algunos elementos secundarios del elemento **activityFeed** que el proveedor OSC debe establecer para que el OSC para mostrar las actividades correctamente en la tarjeta de contacto de Office o el panel de personas de Outlook.</span><span class="sxs-lookup"><span data-stu-id="aa09a-106">The topic also lists a few child elements of the **activityFeed** element that the OSC provider should set for the OSC to display activities properly in the Office Contact Card or Outlook People Pane.</span></span> 
  
- <span data-ttu-id="aa09a-107">El OSC llama al método [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) para obtener y almacenar las actividades en la carpeta de suministro de noticias para el usuario que ha iniciado la sesión.</span><span class="sxs-lookup"><span data-stu-id="aa09a-107">The OSC calls the [ISocialSession2::GetActivitiesEx](isocialsession2-getactivitiesex.md) method to get and store activities in the News Feed folder for the logged-on user.</span></span> <span data-ttu-id="aa09a-108">El proveedor OSC debe implementar **GetActivitiesEx** para devolver una cadena XML de _las actividades_ que cumple con el proveedor de OSC definición del esquema XML del elemento **activityFeed** .</span><span class="sxs-lookup"><span data-stu-id="aa09a-108">The OSC provider must implement **GetActivitiesEx** to return an  _activities_ XML string that complies with the OSC provider XML schema definition of the **activityFeed** element.</span></span> 
    
- <span data-ttu-id="aa09a-109">El proveedor de OSC debe establecer el elemento **ownerID** , que es un elemento secundario del elemento **activityDetails** .</span><span class="sxs-lookup"><span data-stu-id="aa09a-109">The OSC provider must set the **ownerID** element, which is a child element of the **activityDetails** element.</span></span> <span data-ttu-id="aa09a-110">**ownerID** es una cadena opaca que identifica al propietario de la actividad en la red social.</span><span class="sxs-lookup"><span data-stu-id="aa09a-110">**ownerID** is an opaque string that identifies the owner of the activity on the social network.</span></span> 
    
- <span data-ttu-id="aa09a-111">El proveedor de OSC debe establecer los elementos **nameHint** y **emailAddress** en el nodo **publisherVariable** del elemento **templateVariables** .</span><span class="sxs-lookup"><span data-stu-id="aa09a-111">The OSC provider should set the **nameHint** and **emailAddress** elements in the **publisherVariable** node of the **templateVariables** element.</span></span> 
    
   <span data-ttu-id="aa09a-112">Tenga en cuenta que, según el esquema XML del proveedor OSC, el elemento **nameHint** es un elemento opcional.</span><span class="sxs-lookup"><span data-stu-id="aa09a-112">Note that per the OSC provider XML schema, the **nameHint** element is an optional element.</span></span> <span data-ttu-id="aa09a-113">El OSC lo usa para que coincida con el nombre para mostrar del usuario seleccionado en la tarjeta de contacto o el panel de personas.</span><span class="sxs-lookup"><span data-stu-id="aa09a-113">The OSC uses it to match the display name of the user selected in the Contact Card or People Pane.</span></span> <span data-ttu-id="aa09a-114">De forma similar, el elemento **emailAddress** es un elemento opcional en el esquema XML.</span><span class="sxs-lookup"><span data-stu-id="aa09a-114">Similarly, the **emailAddress** element is an optional element in the XML schema.</span></span> <span data-ttu-id="aa09a-115">El OSC lo usa para que coincida con la dirección SMTP del usuario seleccionado en la tarjeta de contacto o el panel de personas.</span><span class="sxs-lookup"><span data-stu-id="aa09a-115">The OSC uses it to match the SMTP address of the user selected in the Contact Card or People Pane.</span></span> 
    
   <span data-ttu-id="aa09a-116">Si sólo se especifica el elemento **ownerID** , pero no se especificaron uno o ambos de **nameHint** y **emailAddress** , el OSC llama al método de [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) y, a continuación, en la [ISocialPerson::GetDetails](isocialperson-getdetails.md) método para obtener más información acerca de la persona identificada por el **ownerID**.</span><span class="sxs-lookup"><span data-stu-id="aa09a-116">If only the **ownerID** element is specified, but one or both of **nameHint** and **emailAddress** are not specified, the OSC calls the [ISocialSession2::GetPeopleDetails](isocialsession2-getpeopledetails.md) method and then the [ISocialPerson::GetDetails](isocialperson-getdetails.md) method to get more information about the person identified by the **ownerID**.</span></span> <span data-ttu-id="aa09a-117">Cuando el OSC llama a **ISocialPerson::GetDetails**, el proveedor debe devolver **persona** XML que especifica los elementos **fullName** y **emailAddress** .</span><span class="sxs-lookup"><span data-stu-id="aa09a-117">When the OSC calls **ISocialPerson::GetDetails**, the provider must return **person** XML that specifies the **fullName** and **emailAddress** elements.</span></span>
