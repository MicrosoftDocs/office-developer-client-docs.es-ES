<span data-ttu-id="8e839-p101">Para comandos no parametrizados, durante la ejecución del comando, se ejecutan todos los comandos de proveedor y se crean los **conjuntos de registros**. Si el comando se ejecuta sincrónicamente, todos los **conjuntos de registros** se llenarán completamente. Si se seleccionó un modo de llenado asincrónico, el estado relleno de los **conjuntos de registros** dependerá del modo de llenado y del tamaño de los **conjuntos de registros**.</span><span class="sxs-lookup"><span data-stu-id="8e839-p101">For non-parameterized commands, all of the provider commands are executed and the **Recordsets** are created during command execution. If the command is executed synchronously, all of the **Recordsets** will be fully populated. If an asynchronous population mode was selected, the populated state of the **Recordsets** will depend on the population mode and the size of the **Recordsets**.</span></span>

<span data-ttu-id="8e839-107">Por ejemplo, el *comando a principal* podría devolver un **conjunto de registros** de clientes para una compañía desde una tabla Customers y el *comando a secundario* podría devolver un **conjunto de registros** de pedidos para todos los clientes desde una tabla Pedidos.</span><span class="sxs-lookup"><span data-stu-id="8e839-107">For example, the *parent-command* could return a **Recordset** of customers for a company from a Customers table, and the *child-command* could return a **Recordset** of orders for all customers from an Orders table.</span></span>

```vb 
 
SHAPE {SELECT * FROM Customers} 
 APPEND ({SELECT * FROM Orders} AS chapOrders 
 RELATE customerID TO customerID) 
```

<span data-ttu-id="8e839-108">Para relaciones principal-secundario no parametrizados, cada objeto **Recordset** principal y secundario debe tener una columna en común para poder asociarlos.</span><span class="sxs-lookup"><span data-stu-id="8e839-108">For non-parameterized parent-child relationships, each parent and child **Recordset** object must have a column in common to class="sxs-lookup"><span data-stu-id="8e839-109">The columns are named in the RELATE clause, *parent-column* first and then *child-column*.</span></span> <span data-ttu-id="8e839-110">Las columnas class="sxs-lookup"><span data-stu-id="8e839-110">The columns may have different names in their respective **Recordset** objects but must refer to the same information in order to specify a meaningful relation.</span></span> <span data-ttu-id="8e839-111">Por ejemplo, los objetos **Recordset** de **Customers** y **Orders** podrían tener en común un campo customerID de identificación del cliente.</span><span class="sxs-lookup"><span data-stu-id="8e839-111">For example, the **Customers** and **Orders** **Recordset** objects could both have a customerID field.</span></span> <span data-ttu-id="8e839-112">Dado que la pertenencia del **conjunto de registros** secundario se determina mediante el comando del proveedor, el **conjunto de registros** secundario puede contener filas <span data-ttu-id="8e839-114">El proceso de forma de datos anexa una columna de capítulo al **conjunto de registros** principal.</span><span class="sxs-lookup"><span data-stu-id="8e839-114">Data shaping appends a chapter column to the parent **Recordset**.</span></span> <span data-ttu-id="8e839-115">Los valores de la columna de capítulo son referencias a filas del **conjunto de registros** secundario, que cumplen la cláusula RELATE.</span><span class="sxs-lookup"><span data-stu-id="8e839-115">The values in the chapter column are references to rows in the child **Recordset**, which satisfy the RELATE clause.</span></span> <span data-ttu-id="8e839-116">Es decir, el mismo valor está en la *columna a principal* de una fila principal determinada y en la *columna a secundaria* de todas las filas del secundario del capítulo.</span><span class="sxs-lookup"><span data-stu-id="8e839-116">That is, the same value is in the *parent-column* of a given parent row as is in the *child-column* of all the rows of the chapter child.</span></span> <span data-ttu-id="8e839-117">Cuando se utilizan varias cláusulas TO en la misma cláusula RELATE, éstas se combinan implícitamente utilizando un operador AND.</span><span class="sxs-lookup"><span data-stu-id="8e839-117">When multiple TO clauses are used in the same RELATE clause, they are implicitly combined using an AND operator.</span></span> <span data-ttu-id="8e839-118">Si las columnas principales de la cláusula RELATE no constituyen una clave para el **conjunto de registros** principal, una fila secundaria única podría tener varias filas principales.</span><span class="sxs-lookup"><span data-stu-id="8e839-118">If the parent columns in the relate clause do not constitute a key to the parent **Recordset**, a single child row may have multiple parent rows.</span></span>
